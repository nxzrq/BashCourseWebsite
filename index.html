The Ultimate Beginner's Guide to Bash Scripting

A comprehensive guide to learning Bash scripting for beginners. Start automating tasks and writing your own powerful scripts.

Chapter 1: Getting Started

What is a Shell? What is Bash?

A shell is a command-line interface (CLI) that allows you to interact with your operating system. You type commands, and the shell executes them.

Bash (Bourne Again SHell) is one of the most popular and widely used shells, especially on Linux and macOS. A Bash script is simply a plain text file containing a series of Bash commands that are executed in order.

Your First Script: Hello, World!

Let's create our first script.

  1. Create the file: Open a terminal and type touch hello_world.sh.
  2. Add the code: Open the file in a text editor and add the following lines:
#!/bin/bash

# This is a comment. The line below will print "Hello, World!"
echo "Hello, World!"

Understand the code:

  • #!/bin/bash (The Shebang): This special line tells the OS which interpreter to use.
  • # (Comments): Lines starting with a # are ignored.
  • echo: A built-in command that prints text to the terminal.

Making a Script Executable and Running It

To run a script, you need to give it "execute" permissions.

  1. Add Execute Permission: chmod +x hello_world.sh
  2. Run the Script: ./hello_world.sh

Chapter 2: Variables and User Input

Variables

Variables store data. Rules: No spaces around =, and access the value with a $ prefix.

#!/bin/bash

# Variable declaration
GREETING="Hello"
USER="testprofile"

# Using the variables
echo "$GREETING, $USER!"

# Using curly braces is a good practice
echo "I am ${USER}."

Command Substitution

You can store the output of a command in a variable.

#!/bin/bash

CURRENT_DIRECTORY=$(pwd)
FILE_LIST=$(ls)

echo "You are currently in: ${CURRENT_DIRECTORY}"
echo "The files here are: ${FILE_LIST}"

Reading User Input with read

Make your scripts interactive by asking the user for input.

#!/bin/bash

read -p "Please enter your name: " NAME
echo "Welcome, ${NAME}!"

Chapter 3: Positional Parameters

Pass arguments to your script from the command line.

  • $0: The name of the script.
  • $1, $2, ...: The arguments.
  • $#: The total number of arguments.
  • $@: All arguments, treated as separate strings.

Save as user_info.sh and run like ./user_info.sh Alice 30.

#!/bin/bash

NAME=$1
AGE=$2

echo "Script name: $0"
echo "Hello, ${NAME}. You are ${AGE} years old."
echo "Total number of arguments: $#"

Chapter 4: Conditional Logic

The if, elif, else Statements

Make decisions in your script. Note the mandatory spaces inside the [ ... ].

#!/bin/bash

read -p "Enter a number: " NUM

if [ "${NUM}" -gt 100 ]; then
  echo "That's a big number."
elif [ "${NUM}" -eq 42 ]; then
  echo "That's the answer to everything!"
else
  echo "That's a small number."
fi

Conditions

String Comparison: ==, !=, -z (empty), -n (not empty)

Integer Comparison: -eq, -ne, -gt, -lt, -ge, -le

File System Checks: -f (is file), -d (is directory), -e (exists)

The case Statement

A cleaner way to handle multiple choices.

#!/bin/bash

read -p "Enter 'start', 'stop', or 'status': " COMMAND

case $COMMAND in
  "start")
    echo "Starting the service..."
    ;;
  "stop")
    echo "Stopping the service..."
    ;;
  "status")
    echo "Checking the status..."
    ;;
  *) # Wildcard for "anything else"
    echo "Unknown command."
    ;;
esac

Chapter 5: Loops

The for Loop

Iterates over a list of items or a range.

#!/bin/bash

# Looping over strings
for PLANET in Mercury Venus Earth Mars; do
  echo "Planet: ${PLANET}"
done

# Looping over a range
for i in {1..5}; do
  echo "Iteration number: ${i}"
done

The while Loop

Continues as long as a condition is true. Uses $((...)) for arithmetic.

#!/bin/bash

COUNTER=1
while [ ${COUNTER} -le 5 ]; do
  echo "Counter is at: ${COUNTER}"
  COUNTER=$((COUNTER + 1))
done

Loop Control: break and continue

break exits the loop. continue skips the current iteration.

#!/bin/bash

for i in {1..10}; do
  if [ ${i} -eq 3 ]; then
    continue # Skip printing 3
  fi
  if [ ${i} -eq 8 ]; then
    break # Stop the loop
  fi
  echo ${i}
done

Chapter 6: Functions

Defining and Calling Functions

Functions are reusable blocks of code.

#!/bin/bash

function greet() {
  echo "Hello from the greet function!"
}

greet

Arguments and Return Values

Functions can take arguments ($1, $2) and you can create local variables. To "return" a value, you echo it and capture it with $(...).

#!/bin/bash

function get_full_name() {
  local FIRST_NAME=$1
  local LAST_NAME=$2
  echo "${FIRST_NAME} ${LAST_NAME}"
}

FULL_NAME=$(get_full_name "John" "Doe")
echo "The full name is: ${FULL_NAME}"

Chapter 7: Practical Example

A simple backup script that combines many of these concepts.

#!/bin/bash

# Function to display how to use the script
function display_usage() {
  echo "Usage: $0 "
}

# --- Main Script Logic ---
if [ $# -eq 0 ]; then
  echo "Error: No directory specified."
  display_usage
  exit 1
fi

DIRECTORY_TO_BACKUP=$1
if [ ! -d "${DIRECTORY_TO_BACKUP}" ]; then
  echo "Error: Directory '${DIRECTORY_TO_BACKUP}' does not exist."
  exit 1
fi

TIMESTAMP=$(date +"%Y-%m-%d_%H-%M-%S")
FILENAME="backup_${TIMESTAMP}_$(basename ${DIRECTORY_TO_BACKUP}).tar.gz"

echo "Backing up '${DIRECTORY_TO_BACKUP}' to '${FILENAME}'..."
tar -czf "${FILENAME}" "${DIRECTORY_TO_BACKUP}"

if [ $? -eq 0 ]; then
  echo "Backup created successfully!"
else
  echo "Error: Backup failed."
fi
